# Scorecard (Metrics & Trends) Flow

## Sequence Diagram

```mermaid
sequenceDiagram
    participant U as User
    participant W as Web Application
    participant B as Backend API
    participant DB as PostgreSQL

    Note over U,DB: Metrics Tab - Page Load

    U->>W: Navigate to /scorecards
    W->>W: Read active team from Zustand

    W->>B: GET /api/v1/weeks
    B->>DB: Query last 13 weeks (ordered by startDate)
    DB->>B: Week list
    B->>W: WeekResponse[]

    W->>W: Determine current week as default

    W->>B: GET /api/v1/metrics?teamId=X&weekId=W
    B->>B: Validate team membership (@PreAuthorize)
    B->>DB: Query metrics for team
    DB->>B: Metric entities
    B->>DB: Query metric values for current + previous week
    DB->>B: MetricValue entities
    B->>B: Map metrics with current value, past value, goal status
    B->>W: MetricResponse[] (with metricValues)
    W->>W: Calculate goal met status per metric
    W->>U: Display scorecard table with goal indicators

    Note over U,DB: Update Weekly Metric Value

    U->>W: Enter value in metric row cell
    W->>B: PUT /api/v1/metric-values {metricId, weekId, value}
    B->>B: Validate team membership via metric
    B->>DB: UPSERT metric_value (insert if not exists, update if exists)
    DB->>B: Updated metric value
    B->>W: MetricValueResponse
    W->>W: Invalidate metrics query cache
    W->>U: Show updated value with goal indicator

    Note over U,DB: Week Navigation

    U->>W: Select different week from dropdown
    W->>B: GET /api/v1/metrics?teamId=X&weekId=W2
    B->>DB: Query metrics for selected week
    DB->>B: Metrics with values for that week
    B->>W: MetricResponse[]
    W->>U: Display scorecard for selected week

    Note over U,DB: Create Metric

    U->>W: Click "Add Metric", fill form
    W->>B: POST /api/v1/metrics {teamId, name, goal, unit, operator, ownerId}
    B->>B: Validate team membership
    B->>B: Validate owner belongs to team
    B->>DB: Insert metric entity
    DB->>B: Saved metric
    B->>W: MetricResponse
    W->>W: Invalidate metrics query cache
    W->>U: Show new metric in scorecard

    Note over U,DB: Update Metric Definition

    U->>W: Edit metric via dialog
    W->>B: PUT /api/v1/metrics/{metricId} {name, goal, unit, operator, ownerId}
    B->>B: Validate team membership
    B->>DB: Update metric fields
    DB->>B: Updated metric
    B->>W: MetricResponse
    W->>U: Show updated metric

    Note over U,DB: Trends Tab

    U->>W: Switch to "Trends" tab
    W->>B: GET /api/v1/metrics/trends?teamId=X
    B->>B: Validate team membership
    B->>DB: Query all metrics for team
    B->>DB: Query metric values across last 13 weeks
    DB->>B: Metric values grouped by metric and week
    B->>B: Build trend data points per metric
    B->>W: TrendsTabMetricListResponse {metrics: [{metricId, metricName, data: [{week, value}]}]}
    W->>W: Render trend table/chart with Recharts
    W->>U: Display weekly trend data across all metrics

    Note over U,DB: My Scorecards (Dashboard Widget)

    U->>W: View dashboard
    W->>B: GET /api/v1/metrics/me?weekId=W
    B->>B: Extract userId from JWT
    B->>DB: Query metrics owned by user (all teams)
    B->>DB: Query metric values for current week
    DB->>B: User's metrics with values
    B->>W: MetricResponse[]
    W->>U: Display personal scorecard widget
```

## Flow Description

1. **Metrics Tab Initialization** - The scorecards page loads available weeks (last 13) and fetches metrics for the active team and current week. Each metric shows its current value, previous week's value, goal, and whether the goal was met.

2. **Week Selection** - Users can navigate between weeks using a dropdown. The last 13 weeks are available, auto-generated by a background scheduler that runs every Monday at 00:01.

3. **Metric Value Entry** - Users enter weekly values directly in the scorecard table. The backend uses an UPSERT pattern (insert if not exists, update if exists) to handle metric values efficiently.

4. **Goal Status Calculation** - Each metric has a goal value and an operator (>=, <=, =, >, <). The goal-met status is calculated by comparing the entered value against the goal using the configured operator. The UI shows green/red indicators.

5. **Multiple Unit Types** - Metrics support five unit types: NUMBER, PERCENTAGE, CURRENCY, YES_NO, and RYG_STATUS (Red/Yellow/Green). The frontend formats values appropriately for each type.

6. **Metric Creation** - New metrics are created with a name, goal, unit type, comparison operator, and assigned owner. The owner must be a member of the team.

7. **Metric Editing** - Metric definitions (name, goal, unit, operator, owner) can be updated without affecting historical values.

8. **Trends Tab** - The trends view shows all team metrics across the last 13 weeks in a tabular/chart format. Data is fetched via a dedicated endpoint that returns pre-grouped trend data points.

9. **Personal Scorecards Widget** - The dashboard shows metrics assigned to the current user with their current week values, providing a quick personal KPI overview.

10. **Background Week Creation** - The `WeekCreationScheduler` runs every Monday at 00:01 to automatically create new week entries (Monday-Sunday range), ensuring the scorecard always has current week slots available.
